# 周报自动化系统

个人用的周例会PPT制作工具。纯静态前端（ESM JS + Supabase），部署在 GitHub Pages。首页（`index.html`）就是核心工作流"制作周报"。

## 制作周报（首页）

选一个目标例会周，页面自动展示"上周总结"（上一个例会周）和"本周计划"（这个例会周），填完直接在本页生成PPT——不用分别去别的页面各选一次目标周。周选择器下方有一个折叠的"+ 新建任务"入口，临时要加一个新任务/新项目不用跳去"任务管理"页面。

想回看/订正更早的周，把目标周改成"那一周的下一周"即可——这跟"总结在下次例会才汇报"的语义是一致的：`weekly_task_entries` 的 `meeting_week_id` 是**任务周**语义，不是"哪次例会上念的"语义，`appears_in='summary', meeting_week_id=N` 表示"第N周计划的完成情况"，即使这份总结实际是在第N+1周例会上口头汇报的。

### 新建任务

完整版新建功能，跟"任务管理"页面的新建表单能力一致——可以新建顺序队列/截止日期项目下的任务、也可以新建循环任务或给已有循环任务生成下一个实例。默认折叠，展开后才会去查一次项目列表。新建成功后，"上周总结"/"本周计划"/"未启动/中止工作"里的搜索候选会立刻包含这个新任务，不用手动点各自的"刷新列表"。

### 上周总结

- "复制上周计划生成总结"：把上周计划的条目复制过来作为起点，正常一周点一次就够；之后如果又给上周计划补录了新条目，可以再点一次——只会补上还没生成过总结的那些，不会覆盖已经填好的行。
- "重点工作完成情况"：从工作群例会纪要粘贴过来的文字，直接进PPT"周工作计划复核情况"表格对应的格子（原样多行显示，不需要额外排版）。
- "记录计划外完成的任务"：没出现在本周计划里、但本周确实做了的任务，搜索一个已有的顺序队列/截止日期/循环任务点击即可添加，"计划内/计划外"自动判定。任务还没建过就用上面的"新建任务"。
- "最终完成情况"会在保存"完成情况"时自动同步——但只有当"本周交付材料"文字跟"最终目标交付物"**严格一致**（去首尾空格）时才会同步成"已完成"；复杂任务允许跨周分批交付，中途标"已完成"只代表这周这部分做完了，"最终完成情况"继续留在"未完成"，任务会继续出现在下周候选池/搜索里，直到哪周的交付材料终于跟最终目标交付物文字对上。这是有意的设计（不加"手动强制标最终完成"的兜底），对不上就是要求交付物文字必须原样一致。
- "风险"这一格有两部分：`风险等级`（低/中/高，决定PPT里这段文字的背景色）和`风险说明`（自由文本，是PPT里实际显示的文字内容）——两者独立，等级只管颜色，说明才是内容。
- "模块"/"责任人"这两列是只读文本，直接显示对应任务自己的值；默认值来自"设置"页面标记的"当前模块"/"当前责任人"。真要改一个任务的模块/责任人，去"任务管理"里改。
- 行顺序默认跟"上周计划"的来源顺序一致，也可以用"↑/↓"手动调整——**PPT就是按这个顺序出表的**，不强制按编号排（比如"上周未完成"经常需要排在"本周新增"前面）。

### 本周计划

- "生成候选池"：按规则自动推荐（顺序队列的当前任务/截止日期到期未完成的任务/本周该出的循环任务实例）。同一个任务如果上周计划里也出现过，计划用时/本周交付物/计划开始/执行截止会自动沿用上周的值；如果是跨多周交付的3级任务且这周不是最终计划完成周，交付物文字末尾"v数字"格式的版本号会自动+1（比如"项目计划书v1"→"v2"，不是这个格式就原样复制）。
- "手动搜索添加任务"：候选池是自动推荐的，可能有遗漏，按编号/标题搜索后点击即可直接加入（默认归类"本周新增"，加入后可以改）。
- "计划开始"/"执行截止"两个日期旁边会显示对应的星期几，方便一眼看出有没有选到假期。
- 同样支持"↑/↓"手动排序，决定PPT里的出表顺序。

### 未启动/中止工作

只列"中止"状态的任务（不列"未启动"——年度计划全量导入后未启动的任务会很多，列进来没意义）。切到一个周会自动搜索一遍所有状态为中止的任务、把还没出现在这周的补进来，不需要手动操作；如果是在"任务管理"页面把某个任务改成中止、而你已经停留在这个页面，点"刷新"重新搜一遍就行。这张表只新增不自动删除——任务如果后来又不是中止状态了，这一行不会自动消失，需要手动点"删除"清理。任务状态本身的改动（标记中止/恢复）都在"任务管理"页面做，不在这里。这张表没有独立的锁定按钮，只读状态跟"本周计划"是否锁定同步。

### 生成PPT

"预览"按钮用跟实际生成PPT完全一样的数据和格式化逻辑，在网页上渲染一张近似的表格（颜色/合并单元格是简化过的，不是逐像素还原），可以先核对内容对不对再下载，省得每次都要打开PPT才发现问题。

### 锁定与订正

计划/总结锁定后所有编辑控件禁用。要改已锁定的数据，先点"解锁编辑"，填一条订正说明才能改（对应PPT"周工作计划复核情况"表格里的备注列）。**这套锁定机制现在对所有任务类型统一生效**——一旦任务的最终目标交付物/最终计划完成时间需要修改，都要走这个订正流程，不只是"进入过周计划的任务"才锁定。

## 任务管理

所有任务（顺序队列/截止日期/循环任务）统一在一棵树里管理，默认全部折叠，只显示1级项目，点▸展开看2级/3级。每一级只显示它自己在这一级的编号（完整编号如"5.1.1"只在制作周报/PPT里用得到）。

- "归属"下拉：选已有项目/循环任务就是给它加一个新任务/生成下一个实例，选"+新建"就是从头建一个。
- 标题/最终目标交付物/最终计划完成时间是每个任务都必填的字段；模块/责任人现在也是必填（默认预填"设置"页面标记的"当前模块"/"当前责任人"）。
- 不管1/2/3哪一级，标题和其它专属设置都要点"详情"才能编辑。
- 二级编号可以留空（代表"项目本身就是任务"，没有再往下分解）；填了三级编号就必须同时填二级编号。二级下如果会挂三级子任务（哪怕只有一条），必须给这个二级填标题。
- 任务一旦分解，WBS编号本身固定不变；"当前执行到第几步"（顺序队列类型）不是存储字段，是按WBS顺序现算第一个未完成任务。
- 删除项目/循环任务会真正释放编号，可以被新项目复用。

## 批量导入

用一份Excel（.xlsx）批量新建当前实际在跟进的项目/任务，不用在"任务管理"一条条手动新建。

- 只导入项目（顺序队列/截止日期两种类型）和它们下面任务的静态结构信息。
- **不导入循环任务**（数量很少，直接去"任务管理"建更快）。
- **也不导入任何一周的历史计划/总结记录**（那是以后自动化历史数据导入要做的事，见 `tools/import/`）。
- 导入前请先确认"设置"页面里已经建好了Excel里会用到的模块名/责任人姓名。
- 模板包含"项目"/"任务"两个sheet，各带一行示例数据。
- 项目编号如果在Excel和数据库里都已存在，会直接复用而不重复创建（所以每周批量导入时，已有项目不需要重新在Excel里列出，只需要在"任务"sheet里引用它的编号）。
- **任务导入不做重复检测**——重新上传同一份Excel修正失败的行之前，请先把已经成功导入的行从Excel里删掉，只保留失败的那些，避免产生重复任务。

## 例会日历

一个自然周 = 周日到周六。默认每周一为例会日/本周工作开始，默认每周五为本周最后工作日；节假日调休导致这两个日期变动时，在表格里手动订正（改完失焦即自动保存）——制定/审核"本周工作计划"时的"计划开始时间"/"执行期"不能填到假期上，就是靠这两个字段圈定范围。

- **整周都没开例会**（比如春节假期）：取消勾选"正常"——下周计划/循环任务实例生成会自动跳过这一周（不留空号，顺延到下一个正常周），但这一周仍保留在日历里。真的要把这一周从日历里彻底抹掉才用"删除"。
- "归属月份"/"月内第几周"是自动算出来的（按例会日所在月份、跳过的周不占号顺延递补），不能手动改，改了例会日或取消勾选"正常"之后会自动重算，可能连带影响同月后面几周的编号。

## 设置

整个app的设置入口，目前只有"模块"和"责任人"两节，以后其它设置也会加在这个页面里。

标了"当前"的模块/责任人会作为"任务管理"新建任务表单、"制作周报"候选池的默认预填值——取代了早期"候选值只有一个时自动选中"的启发式（数量变多后会失效）。

## Supabase 连接诊断

`diagnostics.html`，不在导航栏里，需要时直接访问这个URL——排查数据库连接/登录状态问题用。
